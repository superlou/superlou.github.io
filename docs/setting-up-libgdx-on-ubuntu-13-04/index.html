Setting up libGDX on Ubuntu 13.04

I recently received an Ouya and want to try coding a game for it. While the favorite child for Ouya developments seems to be Unity3D, the Unity IDE cannot be run on Linux. Setup

The libGDX wiki has a list of prerequisites but instead of using standalone Eclipse and installing the Android SDK, we‚Äôre going to use the Android Developer Tools Bundle which contains an IDE (based on Eclipse) with the Android SDK already bundled and configured. You will need to follow the instructions from https://developers.google.com/eclipse/docs/install-eclipse-4.2 to get the version compatible with Eclipse 3.8 which is what ADT is currently built on. If you are using a later version of ADT, you may need to point at a different installation site.

Click next, agree to the licenses, and install.

Download the libGDX SDK from the stable releases. I am using version 0.9.8. I unzipped it into ~/ouya. Set the executable bit on gdx-setup-ui.jar, and run it using a JDK runtime. The GUI should start.
